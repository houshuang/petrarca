1/ Notes on Claude coding after several weeks of heavy use 🧵

The most surprising thing about @AnthropicAI Claude Code isn't the code quality — it's the workflow change. I find myself thinking in higher-level chunks. Instead of "how do I implement this function" I think "what should this module do."

2/ You need to get the context right. I've started maintaining detailed CLAUDE.md files in every project. The better the context, the better the output. It's like writing good documentation but for an AI reader.

3/ My setup: I run 10-15 Claude Code sessions simultaneously. Each session works on a different feature or bugfix. I check in on them periodically, review their work, and merge.

4/ The key insight: treat Claude Code like a junior developer who is incredibly fast but needs clear specifications. Don't vibe code everything — have a plan, break it into pieces, and let the agent execute.

5/ Things that work well:
- Refactoring existing code
- Adding tests to existing functions
- Building new features with clear specs
- Debugging with good error messages

6/ Things that don't work as well:
- Architecture decisions (still needs human judgment)
- Performance optimization (often makes things worse)
- UI/UX design (functional but not beautiful)

7/ Overall: Claude Code has genuinely changed how I write software. The bottleneck has shifted from typing to thinking and reviewing.
